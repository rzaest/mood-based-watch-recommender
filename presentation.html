I started with a Kaggle dataset based on IMDb movies and shows. It had 7,850 rows and useful information such as genres, ratings, votes, and descriptions. But the data was not ready to use. Some descriptions were missing, some were very short, and a more serious problem was that some descriptions did not match the correct movie. If the description is wrong, the recommendation will also be wrong.

My cleaning process had four main steps. First, I searched for missing, repeated, very short, or suspicious descriptions. Second, I checked the IMDb ID together with the title, year, and genre, so I would not mix up films with similar names. Third, I used verified Wikipedia text when a description needed to be improved. Finally, I checked the cleaned file again. The final dataset has 7,848 usable titles and no missing descriptions.

In simple words: the computer turns text into numbers, compares those numbers, and finds descriptions with a similar meaning.

This is the full process in simple terms. The user writes what they want. The system looks for the mood and meaning in that request. It compares the request with the descriptions in the dataset and gives a score to each title. The titles with the strongest match appear first. Behind the scenes, text is changed into numbers so that the computer can compare meaning.

I did not start with the most advanced version. My first version used rules, genres, and keywords. It was clear, but it could miss a good film if the exact keyword was not present. The second version compared important words. The current version is meaning-based, so two sentences can still match even when they do not use exactly the same words.

What decides the final order?

Match score How suitable is this title for the request?

Story meaningDoes the description match what the user asked for?
Mood and themesDoes the title have the right feeling and subject?
Genre and formatIs it the kind of content the user wants?
Rating and votesIs there enough evidence that viewers rated it well?
The final score combines these signals instead of trusting only one of them.

The final order is not based on only one thing. Story meaning is the most important part. Mood, themes, and genre help make the result more suitable. Ratings and votes are used as a smaller quality signal. I chose this combination because a title can be popular but still be completely wrong for the user’s mood.

What I learned

Clean and trustworthy data can matter more than using a complicated model.

The first scoring system was fairly straightforward. The most difficult part was the data quality. I had to be careful not to replace a bad description with another wrong one. The biggest lesson for me was that a complicated model cannot fix unreliable data. Cleaning and checking the dataset made the later work much more meaningful.
